About This Dashboard Template

Credits, purpose, and notes on how to configure and extend this HTML-based network dashboard.

Credits

This dashboard template was designed as a reusable, static HTML “start page” for network teams, home labs, and multi-org environments.

Static HTML Local / Git-backed

Primary Use Case

Provide a single, lightweight landing page that:

  • Centralizes quick links for tools and consoles.
  • Organizes visual maps and device groups.
  • Exposes troubleshooting notes and runbooks.
  • Can be cloned and customized per org with minimal edits.

Project Layout (Suggested)

A simple folder structure that keeps the main pages at the root, and groups maps and tools into subfolders: 

build/
  home.html           ← main landing page
  tools.html          ← list of tools
  maps.html           ← maps index (device groups)
  about.html          ← this page
  config.js           ← shared config (org + quick links)

  tools/
    tool-template.html
    <tool-name>.html  ← per-tool detail pages

  images/
    <image-name>.html  ← per-image

  maps/
    office-template/
      office-template-firewalls.html
      map-template.html
    <other-office>/
      ...

You can rename build/ to whatever folder name fits your repo or web server layout. All links are relative.

Shared Configuration (config.js)

The dashboard uses a single JavaScript file to hold org-level settings and quick links. Edit this once per project / environment. 

// config.js
// Global configuration for this dashboard project.

window.DASHBOARD_CONFIG = {
  // Org-level info (used in headers/footers on most pages)
  business: "Test",
  environment: "Test",
  orgName: "Test Org",
  useCase: "Example usage",

  // Quick links for the sidebar (same across all pages)
  quickLinks: [
    { label: "Google", url: "https://www.google.com" },
    { label: "YouTube", url: "https://www.youtube.com" },
    { label: "GitHub", url: "https://github.com" },
  ],
};

// Helper to apply shared config to a given page.
// Call this from the page with a title/subtitle override.

window.applyDashboardConfig = function (pageOptions = {}) {
  const base = window.DASHBOARD_CONFIG || {};
  const cfg = {
    ...base,
    pageTitle: pageOptions.pageTitle || base.pageTitle,
    pageSubtitle: pageOptions.pageSubtitle || base.pageSubtitle,
  };

  // Basic text fields
  const logoEl = document.getElementById("logo-text");
  const envEl = document.getElementById("env-label");
  const orgEl = document.getElementById("org-name");
  const useEl = document.getElementById("use-case");
  const titleEl = document.getElementById("page-title");
  const subtitleEl = document.getElementById("page-subtitle");

  if (logoEl) logoEl.textContent = cfg.business || "Test";
  if (envEl) envEl.textContent = cfg.environment || "Test";
  if (orgEl) orgEl.textContent = cfg.orgName || "Test Org";
  if (useEl) useEl.textContent = cfg.useCase || "Example usage";
  if (titleEl && cfg.pageTitle) titleEl.textContent = cfg.pageTitle;
  if (subtitleEl && cfg.pageSubtitle) subtitleEl.textContent = cfg.pageSubtitle;

  // Quick links in sidebar
  const quickList = document.getElementById("quick-links");
  if (quickList) {
    quickList.innerHTML = "";

    if (Array.isArray(cfg.quickLinks) && cfg.quickLinks.length > 0) {
      cfg.quickLinks.forEach((item) => {
        const li = document.createElement("li");
        const a = document.createElement("a");
        const dot = document.createElement("span");
        const labelSpan = document.createElement("span");

        dot.className = "sidebar-link-dot";
        labelSpan.textContent = item.label;

        a.href = item.url;
        a.target = "_blank";
        a.rel = "noopener noreferrer";

        a.appendChild(dot);
        a.appendChild(labelSpan);

        li.appendChild(a);
        quickList.appendChild(li);
      });
    } else {
      const li = document.createElement("li");
      const span = document.createElement("span");
      span.textContent = "No quick links defined.";
      span.style.fontSize = "0.8rem";
      span.style.color = "var(--text-muted)";
      li.appendChild(span);
      quickList.appendChild(li);
    }
  }
};

Usage & Customization FAQ

How do I change the business / org labels?

Open config.js and edit the DASHBOARD_CONFIG object: 

window.DASHBOARD_CONFIG = {
  business: "MyCompany",
  environment: "Prod",
  orgName: "MyCompany IT",
  useCase: "Network Operations",
  quickLinks: [ /* ... */ ],
};

Most pages read these values automatically via applyDashboardConfig(...).

How do I customize Quick Links?

In config.js, update the quickLinks array: 

quickLinks: [
  { label: "New SolarWinds", url: "https://solarwinds.example.com" },
  { label: "Zabbix",        url: "https://zabbix.example.com" },
  { label: "Secret Server", url: "https://secrets.example.com" },
],

Any page with a sidebar that has id="quick-links" will show the updated list.

How does a page apply the shared config?

At the bottom of each page (except standalone ones like this About page), include the config file and call applyDashboardConfig: 

<!-- Example in home.html (in build/) -->
<script src="config.js"></script>
<script>
  applyDashboardConfig({
    pageTitle: "Network Dashboard",
    pageSubtitle: "Landing page for tools, maps, and docs.",
  });
</script>

For pages in subfolders, adjust the path: <script src="../config.js"> or <script src="../../config.js">.

How do I add a new office / device-group map page?

1. Create a new folder under maps/ or reuse an existing one (e.g. maps/office-template/).
2. Copy an existing map page (like office-template-firewalls.html) as a starting point.
3. Update:

  • The pageTitle / pageSubtitle in the bottom script.
  • The horizontal map navigation links in the .map-nav-strip section.
  • The map image src and Visio / PDF link.
  • The device table (hostnames, roles, IP addresses).

Finally, add a link to this new page from maps.html under the appropriate device group.

How do I add a new tool?

There are two steps:

  1. In tools.html, duplicate the <article class="tool-card"> block and edit:
    • Tool name and description.
    • href="tools/<tool-name>.html" for the “More details” link.
    • The “Open” link URL (typically a web UI).
  2. Copy tools/tool-template.html to tools/<tool-name>.html and customize:
    • Page title and subtitle.
    • Summary text and tags.
    • Primary site, access details, and documentation links.
Why doesn’t the dashboard run EXE files directly?

Browsers intentionally prevent web pages from directly running local executables for security reasons. Links to local paths (like file:// or C:\...) are treated as files to view or download, not to execute.

For “desktop shortcut” behavior, use:

  • Web URLs for web consoles (preferred).
  • Documented UNC paths (e.g. \\server\share\tool).
  • OS-managed shortcuts or custom protocol handlers (advanced, outside this static HTML template).

Notes

This template intentionally avoids build systems, frameworks, and heavy dependencies. It is designed to live comfortably in:

  • A simple Git repo synced between team members.
  • A basic internal web server (NGINX, Apache, IIS, etc.).
  • Local files opened directly in a browser (with relative paths).

Feel free to modify as needed for the requirements of your team or environment. For reference, I primairly use this through git as a simple html page. Last thing, the best notes aren't in the FAQ; they're in the comments of the code!
